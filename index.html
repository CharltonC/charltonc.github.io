Draft - React state management (library / 2024)

Extremely simple and efficient OOP style state management for React

  • event handler method names directly represents action, no need for ceremonial redux actions
  • update state (cloned) directly inside wrapped methods with optional stackable update via this.reflect (i.e. access method without state update)
  • allows async for event handler for any api call
  • compare and skip state update if same
  • optional integration with 3rd-party or external modules initialization inside constructor without useEffect
  • optionally exclude event handlers from being wrapped with state update
  • init once and use method directly in event, e.g. onClick={onBtnClick} instead of onClick={() => onBtnClick() }
