Draft - React state management (library / 2024)

Extremely simple and efficient OOP style state management for React

  • event handler method names directly represents action, no need for ceremonial redux actions
  • allows async for event handler for any api call
  • directly update state (cloned) inside wrapped methods with optional stackable update via `this.reflect.method` (i.e. original unwrapped method but with state argument)
  • state update integration with initialization of 3rd-party/external modules inside `constructor` without `useEffect`
  • state comparison and skip update if same
  • flexibly include/exclude event handlers from being wrapped with state update
  • init once and use method directly in event, e.g. `onClick={onBtnClick}` instead of `onClick={() => onBtnClick() }`
